# Running Tests

Learn how to run load tests and get comprehensive performance analysis.

### Quick Test
```bash

# Run with specific configuration

```

### Extended Test
```bash
# Run a 15-minute test with 50 virtual users
python run_test.py configs/pop_website_15min_test.yaml
```

## ğŸ“Š What Happens During a Test

1. **Configuration Loading** - YAML config is parsed and validated
2. **Docker Build** - k6 container is built (first time only)
3. **Load Test Execution** - k6 runs the performance test
4. **Results Collection** - Metrics are collected and organized
5. **Analysis Pipeline** - Page resources and performance are analyzed
6. **AI Analysis** - OpenAI generates optimization recommendations

## ğŸ¯ Test Configurations

### Available Configurations

## ğŸ“ Results Organization

### Automatic Organization
Results are automatically organized by:
- **Site name** (extracted from URL)
- **Timestamp** (YYYYMMDD_HHMMSS format)
- **Latest symlink** (always points to most recent)

### Result Files
Each test run creates:
- `summary.json` - Raw k6 metrics
- `test_report.json` - Formatted summary
- `page_resource_analysis.json` - Page analysis
- `enhanced_analysis_report.json` - Performance analysis
- `ai_analysis_report.json` - AI recommendations

## ğŸš¨ Troubleshooting

### Common Issues

**Docker not running:**
```bash
# Start Docker Desktop
open -a Docker  # macOS
```

**Permission errors:**
```bash
# Ensure Docker permissions
sudo usermod -aG docker $USER
```

**OpenAI API errors:**
```bash
# Check API key
cat .env | grep OPENAI_API_KEY
```

### Test Timeouts
- **Default timeout**: 5 minutes
- **Extended tests**: May need longer timeouts
- **Monitor progress**: Watch console output
